HIGH — 需要优先处理

#1 normalizePostSlug 函数重复定义

代码重复
astro.config.mjs:13-18 & src/utils/slug.ts:8-14
两个文件中的 normalizePostSlug 实现完全一致，执行相同的 trim、正则替换和 slug 规范化逻辑。
风险：修改 slug 规则时需同步改两处，遗漏将导致 sitemap 中的 URL 与实际路由不一致。

#2 127 个工具组件重复相同的"独立打开"按钮代码

代码重复架构
src/components/tools/*.astro（全部 127 个文件）
每个工具组件都内联了相同的 Props 接口定义和"独立打开"按钮模板（含 SVG 图标和样式类）。
interface Props {
  cleanPath?: string;
  showCleanLink?: boolean;
  showPageMeta?: boolean;
}
// + 完全相同的 <a> 按钮模板，127 份拷贝
风险：改按钮样式或行为需改 127 个文件。Props 接口也应提取为共享类型。

#3 QR 码生成器功能损坏

功能缺陷
src/components/tools/QrGenTool.astro:16-111
QR 码生成器未实现真正的 QR 编码算法，而是用随机种子生成类似 QR 码的图案。代码注释明确写着 "This generates a visual QR-like pattern"。
风险：用户生成的"QR 码"无法被任何扫描器识别，严重影响工具可信度。

#4 ai-news.json 巨型单文件（144,000+ 行）

性能
src/data/ai-news.json（4,197 条目）
AI 资讯数据以单个 JSON 文件存储，每次构建全量加载解析，且包含每条新闻的完整内容文本。
风险：构建时间和内存消耗随数据增长线性上升。文件只增不减。

#5 零测试覆盖

质量DevOps
项目全局
项目中没有任何测试文件（*.test.*、*.spec.*、__tests__/），package.json 中也没有 test 脚本。
风险：所有逻辑变更只能通过构建验证，无法发现运行时行为回归。工具组件的复杂客户端逻辑尤其需要测试。

MEDIUM — 建议尽快处理

#6 escapeHtml 在 3 个工具组件中重复实现

代码重复安全
RegexTool.astro:169 / TextDiffTool.astro:171 / JsonCsvTool.astro:416
标准实现位于 src/utils/safeHtml.ts:3，部分工具（如 RegexVisualizerTool、SplitBillTool）已正确导入，但以上 3 个工具在 script 块中重新实现了本地版本。
风险：本地版本可能缺少转义规则（如单引号、反引号），存在 XSS 隐患。

#7 点赞按钮逻辑重复实现

代码重复
src/pages/posts/[...slug].astro:320-365 & src/components/PageLikeButton.astro:29-84
文章详情页内联了自己的点赞按钮实现，未复用已有的 PageLikeButton 组件，两者独立维护图标状态和 .liked 样式。
风险：两套逻辑需同步修改，行为可能产生差异。

#8 分页逻辑在 3 个文件中各自实现

代码重复
src/pages/posts/index.astro / src/pages/posts/page/[page].astro / src/components/AINewsArchivePage.astro
页码生成（含省略号处理、前/后页链接）在三处独立实现，且行为有差异（首页用简化版）。
风险：分页 UX 不一致，维护成本高。

#9 工具页意外显示文章评论和点赞功能

架构
src/layouts/MainLayout.astro:44-46
MainLayout 中 showSubpageFeedback 判断让所有 /tools/X 路径都显示 Giscus 评论和点赞按钮——这些功能本意是给博客文章的。
风险：工具页出现无关评论区，影响用户体验。

#10 254 个工具页面可用动态路由合并

架构
src/pages/tools/（254 个文件）
每个工具有两个静态路由：/tools/X.astro（带站点壳）和 /tools/X/clean.astro（独立版）。127 个工具 × 2 = 254 个页面。
风险：静态路由数量膨胀，构建产物体积大，新增工具需创建两个文件。

#11 文章详情页过于庞大（366 行）

架构
src/pages/posts/[...slug].astro
单文件包含：静态路径生成、文章头部、内容渲染、内联点赞按钮、上下篇导航、相关文章、作者简介、内联样式、完整客户端脚本。
风险：职责过多，难以定位和修改特定功能。点赞按钮应复用组件。

#12 sync-ai-news.mjs 单文件 1594 行

架构
scripts/sync-ai-news.mjs
一个文件混合了 RSS 抓取、内容提取、翻译、图片下载、评分、去重、JSON 输出全部逻辑。
风险：难以单独测试或修改某个环节，改动容易引发连锁问题。

#13 大量 innerHTML 使用，转义不一致

安全
89 处分布在 src/components/tools/*.astro
多数工具组件通过 innerHTML 渲染动态内容，虽然大部分使用了 escapeHtml()，但存在未转义或转义不完整的案例（如 ColorPickerAdvancedTool:54、MemoryMatchTool:59、FlexboxPlaygroundTool:38）。
风险：一旦某个工具遗漏转义，即构成 XSS 漏洞。模式本身就很脆弱。

#14 CI/CD 缺少代码质量检查

DevOps
.github/workflows/deploy.yml
部署工作流只有 npm ci → npm run build → wrangler deploy，没有 lint、typecheck 或 test 步骤。代码质量完全依赖本地检查。
风险：类型错误、代码风格问题可直接部署到生产环境。

#15 Workers CORS 配置为通配符 *

安全
workers/wrangler.toml:13
ALLOWED_ORIGINS = "*" 允许任何来源的跨域请求访问统计 API。
风险：任何网站都可以调用统计接口刷阅读量/点赞数。应限制为已知域名。

#16 ai-news-sync.yml 使用 require()

DevOps
.github/workflows/ai-news-sync.yml:40
项目 package.json 声明 "type": "module"，但 CI 的 inline node 脚本使用 require('./src/data/ai-news.json')。Node.js 在 ESM 模式下不支持 require。
风险：CI 步骤可能静默失败或抛出错误，导致同步摘要不显示。

#17 as any 类型断言

类型安全
PageLikeButton.astro:73-74 / PageViewCounter.astro:49-50
两个组件中使用 (window as any)[INIT_FLAG] 来检测初始化状态，绕过了 TypeScript 类型检查。
风险：虽然功能无碍，但破坏了类型安全，且模式可在 @utils/ 中统一封装。

#18 工具按钮缺少 aria-label

可访问性
src/components/tools/*.astro 大部分文件
127 个工具组件中的功能按钮（如"编码"、"解码"、"复制"、"清空"等）绝大多数没有 aria-label，仅依赖按钮文字。部分按钮只含 SVG 图标无文字（如删除按钮）。
风险：屏幕阅读器用户无法理解纯图标按钮的功能。

#19 Base64ImageTool img 缺少 alt 属性

可访问性
src/components/tools/Base64ImageTool.astro:20
<img id="img" class="max-w-full max-h-96 mx-auto rounded" /> 缺少 alt 属性。虽然这是动态工具，但仍应提供空 alt="" 或动态 alt。

#20 无 Content Security Policy 安全头

安全
项目全局（Cloudflare Pages 配置）
站点加载了外部脚本（Giscus 评论、Google AdSense），但没有配置 CSP 头来限制脚本来源、样式来源和内联脚本。
风险：如果任何注入点被利用，攻击者可加载任意脚本。可在 Cloudflare Pages 的 _headers 或 Workers 中配置 CSP。

LOW — 可在空闲时处理

#28 tsconfig path alias 未完全统一使用

#29 sync-cnblogs 缺少超时和中断处理

健壮性
scripts/sync-cnblogs.mjs
网络请求（RSS 抓取、图片下载）没有设置超时或 AbortController。在 CI 环境中，网络问题可能导致脚本永久挂起，直到 workflow 30 分钟超时。

#30 外部脚本无 SRI 校验

安全
src/layouts/BaseLayout.astro:196 / src/components/GiscusComments.astro:29-30
Google AdSense (pagead2.googlesyndication.com) 和 Giscus (giscus.app/client.js) 的外部脚本均未设置 integrity 和 crossorigin 属性（SRI）。
风险：如果 CDN 被入侵，可执行任意代码。SRI 可确保脚本内容未被篡改。
